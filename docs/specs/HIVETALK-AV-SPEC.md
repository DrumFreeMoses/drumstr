# HIVETALK-AV-SPEC — Self-Hosted A/V Server (HiveTalk / MiroTalk SFU)

## 1. Project Details

| Field | Value |

## 2. End Goal

A self-hosted WebRTC A/V server running on `av.drumstr.app` capable of hosting ≥ 100 simultaneous audio/video participants per session. Drumstr users join group drumming rooms created by the backend API. HiveTalk (Nostr-native) is the primary choice; MiroTalk SFU is the fallback. The room URL is generated by the backend and delivered to clients.

## 3. Current Implementation Details

Repository initialized at `~/repos/drumstr`. No A/V server deployed yet. MiroTalk SFU documentation reviewed at `docs.mirotalk.com/mirotalk-sfu/self-hosting/`. Hetzner A/V VPS not yet purchased.

## 5. Current Proposed Solution

### Primary Choice: HiveTalk (Nostr-native)
- HiveTalk integrates Nostr identities directly — users auth with Nostr keys.
- Self-hosted via Docker.
- Exposes a room URL that clients open via WebView (mobile) or iframe/redirect (web).
- **Issue:** Self-hosting documentation was inaccessible during spec creation. Research required.

### Fallback: MiroTalk SFU
- Proven, well-documented WebRTC SFU.
- Uses same `av.drumstr.app` domain.
- Rooms created by API call; room URLs passed to clients.

### Decision for v1: **MiroTalk SFU** (proceed with documented fallback; integrate HiveTalk when docs available)

### Capacity Planning (100 A/V Users)
| Parameter | Value |
|---|---|
| Users per session | Up to 100 |
| Ports per user | 2 (1 audio + 1 video) |
| Port range | 40000–40200 (TCP + UDP) |
| Recommended vCPU | 4 (SFU_NUM_WORKERS=4) |
| Recommended RAM | 8GB |
| Estimated bandwidth | ~2 Mbps/user × 100 = ~200 Mbps |
| Hetzner CX31 included traffic | 20TB/mo — sufficient |

### Architecture

```
Client (browser/mobile)
        │  WebRTC (UDP 40000-40200)
        ▼
av.drumstr.app (Cloudflare DNS-only, grey cloud)
        │  HTTPS 443
        ▼
Nginx (SSL termination via Certbot)
        │  HTTP → localhost:3010
        ▼
MiroTalk SFU (Node.js, port 3010)
        │
        ├── Room API: POST /api/v1/meeting (create named room)
        └── Room URL: https://av.drumstr.app/join?room={roomId}
```

### Nginx Config (av.conf)
```nginx
server {
    listen 80;
    server_name av.drumstr.app;
    return 301 https://$host$request_uri;
}
server {
    listen 443 ssl;
    server_name av.drumstr.app;
    ssl_certificate /etc/letsencrypt/live/av.drumstr.app/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/av.drumstr.app/privkey.pem;

    location / {
        proxy_pass http://localhost:3010;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

### MiroTalk SFU .env
```bash
ENVIRONMENT=production
SFU_ANNOUNCED_IP=<A/V VPS public IP>
SFU_MIN_PORT=40000
SFU_MAX_PORT=40200
SFU_NUM_WORKERS=4
# Restrict API access — set a strong secret
API_SECRET=<strong-random-secret>
```

### Room Creation API (called by Backend API)
```http
POST https://av.drumstr.app/api/v1/meeting
Authorization: Secret <API_SECRET>
Content-Type: application/json

{ "room": "drumstr-event-{eventId}" }

Response: { "meeting": "https://av.drumstr.app/join?room=drumstr-event-{eventId}" }
```

## 6. Next Steps

1. **Research HiveTalk self-hosting**
   - Visit https://github.com/hivetalk and https://hivetalk.org
   - Determine if HiveTalk supports self-hosting and what the Docker setup looks like
   - If viable, create `HIVETALK-SELF-HOST-RESEARCH.md` with findings

2. **Provision A/V VPS on Hetzner**
   - Create CX31 server (Ubuntu 22.04 LTS) in Hetzner Cloud console
   - Add SSH public key, note public IP

3. **Run bootstrap-av.sh** (from INFRA-SPEC)
   - Installs Docker, Node.js, ffmpeg, configures firewall

4. **Clone and configure MiroTalk SFU**
   ```bash
   git clone https://github.com/miroslavpejic85/mirotalksfu.git /opt/mirotalksfu
   cd /opt/mirotalksfu
   cp .env.template .env
   # Edit .env: set ENVIRONMENT, SFU_ANNOUNCED_IP, port range, SFU_NUM_WORKERS=4, API_SECRET
   cp app/src/config.template.js app/src/config.js
   ```

5. **Install dependencies and start**
   ```bash
   cd /opt/mirotalksfu
   npm install
   # Install PM2
   npm install -g pm2
   pm2 start server.js --name mirotalksfu
   pm2 save && pm2 startup
   ```

6. **Obtain SSL certificate**
   ```bash
   apt install certbot python3-certbot-nginx
   certbot --nginx -d av.drumstr.app
   ```

7. **Configure Nginx** — place `av.conf` in `/etc/nginx/conf.d/`, test and reload:
   ```bash
   nginx -t && systemctl reload nginx
   ```

8. **Verify room creation** — test the API endpoint from the backend server:
   ```bash
   curl -X POST https://av.drumstr.app/api/v1/meeting \
     -H "Authorization: Secret <API_SECRET>" \
     -H "Content-Type: application/json" \
     -d '{"room": "test-room-1"}'
   ```

9. **Test with 100 simulated users** — use a WebRTC load testing tool (e.g., testrtc.com or selenium-based script) to verify capacity before launch.

10. **Document room URL format** in `packages/shared/src/constants.ts`:
    ```typescript
    export const AV_ROOM_BASE_URL = process.env.HIVETALK_URL ?? 'https://av.drumstr.app'
    export const getRoomUrl = (eventId: string) =>
      `${AV_ROOM_BASE_URL}/join?room=drumstr-event-${eventId}`
    ```
